# Security Policy

## Reporting a Vulnerability

We take the security of TaskBridge seriously. If you believe you've found a security vulnerability, do not open a public issue.

Use one of these private reporting routes:

1. Prefer the GitHub Security Advisory flow if the repository Security tab shows a `Report a vulnerability` button.
2. If private reporting is not yet enabled, contact the maintainer through the public address listed on the GitHub profile or repository metadata and include `TaskBridge security report` in the subject.
3. Include a description of the issue, affected versions, reproduction steps, and any suggested mitigation.

We will acknowledge receipt and coordinate a fix timeline before public disclosure.

## Supported Versions

Only the latest version of TaskBridge is supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |
